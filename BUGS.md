# Vault Native — Bug & Issue Audit

| # | Severity | Issue | Where | Description |
|---|----------|-------|-------|-------------|
| 1 | ~~**CRITICAL**~~ **FIXED** | Shallow merge loses sites on Nostr restore | `app.js:820,833` | **Fixed:** replaced shallow spread with deep-merge loop (higher nonce wins) in both `tryApplyBackup` code paths. |
| 2 | ~~**CRITICAL**~~ **FIXED** | `restoreFromId` replaces instead of merging | `app.js:2307` | **Fixed:** replaced full replacement with guarded deep-merge loop (higher nonce wins). |
| 3 | ~~**CRITICAL**~~ **FIXED** | `restoreFromNostr` also shallow-merges | `app.js:2147` | **Fixed:** replaced shallow spread with deep-merge loop (higher nonce wins) in manual restore path. |
| 4 | ~~**HIGH**~~ **FIXED** | CSP mismatch — Tauri allows `script-src 'unsafe-inline'` | `tauri.conf.json:12` | **Fixed:** removed `'unsafe-inline'` from `script-src` in `tauri.conf.json`. `style-src 'unsafe-inline'` intentionally kept (inline `<style>` block, low risk). |
| 5 | ~~**HIGH**~~ **FIXED** | No master password minimum length | `app.js:1392` | **Fixed:** added `MIN_PASSWORD_LENGTH = 8` constant and validation in both `setMasterPassword()` and `showBackupPasswordModal()` (set/change modes only — existing short passwords can still unlock). |
| 6 | ~~**HIGH**~~ **FIXED** | CryptoJS AES uses weak key derivation (EVP_BytesToKey/MD5) | `app.js` | **Fixed:** replaced CryptoJS.AES with Web Crypto API (PBKDF2-SHA256, 600k iterations, AES-256-GCM) for `saveLocalNonceBackup()` and `autoSaveVault()`. Legacy CryptoJS decrypt preserved for migration — on first unlock, data is re-encrypted with Web Crypto. Derived key cached in memory per session. |
| 7 | ~~**HIGH**~~ **FIXED** | `connectRelay` leaks WebSocket on timeout | `app.js:1718` | **Fixed:** added `relay.close()` before `reject()` in both the timeout and error handlers of `connectRelay()`. |
| 8 | ~~**HIGH**~~ **FIXED** | Lock-without-password UX is confusing | `app.js:918` | **Fixed:** extended `showConfirm()` with custom button labels and danger styling. Replaced two confusing dialogs with a single dialog: "Set Password" (primary) / "Lock Anyway" (danger red). |
| 9 | MEDIUM | Service worker cache update silently fails | `sw.js:39-43` | `cache.put(event.request, response)` consumes the response body, then `response.clone()` is called after consumption. The correct pattern is `cache.put(event.request, response.clone())`. The user gets the stale cached version, but the cache never actually updates in the background — app updates won't propagate until the service worker is reinstalled. |
| 10 | MEDIUM | `_backupDebounceTimer` not cleared on lock/delete | `app.js:884-940,1377-1398` | `lockVault()` clears `inactivityTimer` and `clipboardClearTimer`, but not `_backupDebounceTimer`. If a backup was queued, `setRelayStatus('syncing')` was already called (line 1303), leaving the relay indicator stuck on "syncing" after lock with no resolution. The backup itself aborts (checks `vault.privateKey`), but the UI state is stale. |
| 11 | MEDIUM | `popstate` handler skips screen-specific setup | `app.js:2579-2587` | The browser/Android back button handler directly toggles CSS classes instead of calling `showScreen()`. Screen-specific setup is skipped — e.g., landing on `mainScreen` via back button won't call `renderSiteList()`, so the site list may be stale or empty. |
| 12 | MEDIUM | HTML CSP missing `'self'` in `connect-src` | `index.html:5` | `connect-src` lists only relay URLs and `https://*.github.io` but omits `'self'`. In pure browser/PWA mode, fetch requests to the app's own origin are blocked. Works fine in Tauri (where `tauri.conf.json` CSP applies), but breaks the PWA fallback. |
| 13 | MEDIUM | Race condition: remote restore during active use | `app.js:750-772` | `checkForRemoteBackups()` runs asynchronously after vault init. If the user starts adding sites while the loading modal is up, the remote restore could complete and overwrite them via the shallow merge (#1). The loading modal mitigates this somewhat, but doesn't fully block interaction. |
| 14 | LOW | `generateNewSeed` sets `vault.seedPhrase` before verification | `app.js:572` | The seed phrase is stored in `vault.seedPhrase` as soon as it's generated, before the user verifies they've backed it up. If they navigate away without completing verification, an unverified seed is in memory and could be acted upon. |
| 15 | LOW | `escapeJsString` is defined but never used | `app.js:1081-1083` | Dead code. Can be removed. |
| 16 | LOW | `toggleDebugMode` triggers Nostr backup | `app.js:1601` | Toggling debug mode (a UI preference) calls `backupToNostrDebounced()`, publishing to relays for a cosmetic setting change. Unnecessary network traffic and relay writes. |
| 17 | LOW | Seed words rendered via `innerHTML` without escaping | `app.js:580,1637` | `div.innerHTML = \`<span>${i + 1}.</span>${word}\`` — BIP39 words are safe (a-z only), so this isn't exploitable in practice, but it breaks the escaping convention used elsewhere (`escapeHtml` in `renderSiteList`). |
| 18 | LOW | No validation on nonce values from imports/backups | `app.js:1551-1558` | Nonces from external sources (JSON import, Nostr restore) are never validated as non-negative integers. A malformed backup with `nonce: "abc"` or `nonce: -1` would be accepted and could break nonce increment/decrement logic. |
